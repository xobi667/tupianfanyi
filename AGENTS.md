# AGENTS.md

## 项目规则

- 当前项目是本地图片翻译器，根目录为 `E:\图片翻译器`，真实应用在 `project/`。
- 不要删除 `资源/`，这里保存用户原图、翻译结果、历史清单和日志。
- 不要把 API 秘钥写进源码、Markdown、历史记录或日志。
- 默认端口是 `3006`。

## 编码安全

- 所有源码、JSON、Markdown、HTML、CSS、TS、TSX、JS 文件默认按 UTF-8 处理。
- 修改中文/CJK 文案时必须保留中文，不允许替换成 连续问号、乱码、拼音或占位文本。
- 写文件优先使用 Python `Path.read_text/write_text(encoding="utf-8")` 或明确 UTF-8 的写法。
- 不要用 PowerShell 默认 `Set-Content` 写中文，除非显式 `-Encoding UTF8` 并确认输入没坏。
- 不要全局替换 `??`，它可能是 TypeScript 的空值合并运算符。

## UI 规则

- 空页面只做全屏上传入口，不显示设置、历史、语言、比例、并发等工作台控件。
- 上传后才显示工作台完整控制。
- 图片墙必须尽量吃满宽度，不要在右侧留大块空白。
- 图片预览使用方形容器，图片完整显示，避免裁切。
- 右侧控制栏默认收起，只在鼠标碰到最右侧时覆盖弹出，不要挤压布局。
- 单图重新翻译/重新重绘必须只处理当前图片。

## 历史与数据安全

- 上传原图后应立即保存到 `资源/`。
- 翻译结果生成后应立即保存到 `资源/`。
- 刷新或恢复历史前，要先保护当前工作台状态。
- 删除历史任务后，如果还有下一个任务，应自动显示下一个任务详情。
- 删除任何本地图片或历史任务前要确认这是用户明确触发的删除行为。

## 验证要求

改动后至少运行：

```bash
cd project
npm run lint
npx tsc --noEmit
```

中文/乱码扫描：

```text
检查 app/lib/scripts 中是否出现：连续问号占位符、U+FFFD replacement character、常见中文 mojibake 片段。
不要把这些坏字符字面量写进项目文档，否则会造成扫描误报。
```
