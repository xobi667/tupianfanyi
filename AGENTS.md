# AGENTS.md

## 项目规则

- 项目根目录：`E:\图片翻译器`。
- 真实应用目录：`project/`。
- 默认端口：`3006`。
- 品牌名：`xobi`。
- 目标用户：个人高频使用图片翻译工作台，不是公开 SaaS 后台。

## 绝对不要做

- 不要删除 `资源/`，里面是用户原图、结果图、历史和日志。
- 不要把 API 秘钥写进源码、Markdown、历史记录或日志。
- 不要用浏览器原生 `confirm/alert` 做主要 UX。
- 不要把暂停后的“继续”做成重新翻译。
- 不要把中文写坏成问号、乱码、HTML 实体、拼音或占位符。
- 不要全局替换 `??`，它可能是 TypeScript 空值合并运算符。

## 编码安全

- 所有源码、JSON、Markdown、HTML、CSS、TS、TSX、JS 文件默认 UTF-8。
- 修改中文/CJK 文案前先确认文件能按 UTF-8 正常读取。
- 写文件优先用 Python `Path.read_text/write_text(encoding="utf-8")`。
- PowerShell 写中文必须显式 `-Encoding UTF8`，否则不要用。

## UI 规则

- 首页只做上传入口，不显示设置、历史、语言、比例、并发等工作台控件。
- 上传后才显示工作台。
- 图片墙要尽量吃满可用宽度，不在右侧留无意义空白。
- 图片预览要完整显示，不裁切关键内容。
- 鼠标碰右边缘时弹出右侧控制面板，控制面板覆盖显示，不挤压布局。
- 颜色走黑灰工业风 + 翡翠绿/青色/琥珀色，不走粉紫色。
- 动画要有质感，但进度条不能闪得烦。
- 字体、间距、按钮、输入框、下拉框必须统一。

## 交互规则

- 画布支持点击选中、Ctrl/Shift 多选、鼠标框选。
- Delete 是软移除，Ctrl+Z 可以撤回。
- 右键菜单必须尊重选中范围。
- 单图重新翻译/重新重绘只处理当前图或当前选中图，不跑全批。
- 运行中点击“暂停”应立即暂停。
- 暂停后点击“继续”应继续当前未完成任务，不弹“开始翻译”确认，不重复成功图片。
- 开始新批次时可弹 xobi 自定义确认框，里面能改语言和比例。

## 历史和数据安全

- 上传原图后应立即保存到 `资源/`。
- 翻译结果生成后应立即保存到 `资源/`。
- 刷新或恢复历史前要保护当前工作台状态。
- 删除历史任务后，如果还有下一个任务，应自动显示下一个任务详情。
- 删除任何本地图片或历史任务前要确认这是用户明确触发。

## 验证要求

代码改动后至少运行：

```bash
cd project
npm run lint
npm run build
```

安全/依赖检查：

```bash
npm audit --audit-level=moderate
```

MCP 浏览器检查优先项：

- 首页能打开。
- 上传后进入工作台。
- 右侧 hover 控制面板正常。
- 暂停后继续不弹确认框。
- 历史和设置弹层能打开。
- 控制台没有明显错误。
